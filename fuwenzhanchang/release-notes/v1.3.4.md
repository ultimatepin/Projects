# Rift Local v1.3.4

v1.3.4 rebuilds the battle screen around the cards, removes repeated controls, and moves routine rule work onto the host so a turn takes fewer clicks.

## Highlights

- A larger, space-aware hand uses the available width and wraps only when the card count requires it.
- The fixed action drawer is replaced by compact inline actions beside the selected card.
- Empty lanes collapse and player/Rune summaries take less room, leaving more of the screen for cards.
- Playing a card now automatically exhausts and recycles the required Runes in one atomic server action.
- The play preview shows the automatic Rune plan before you commit.
- Redundant Rune-deck controls and the in-battle Core Rules panel are removed.

## Rules and card actions

- Card costs can no longer be bypassed by omitting or changing the client payment payload; the host derives and validates payment from the card and live Rune state.
- Discard and recycle remain effect-bound actions instead of unrestricted buttons.
- Zaun Warrens now implements its Conquer trigger: discard 1, then draw 1.
  - The trigger first enters a Focus response window and resolves after both players pass.
  - With no card in hand, the app skips the impossible discard and draws 1.
  - With exactly one card, it discards that card and draws automatically.
  - With multiple cards, it privately prompts the controller to choose one, then draws automatically.
- Pending private choices pause other game actions until the correct player resolves them.
- A battlefield showdown opened as a response resolves first, then resumes the original trigger window with the correct player holding Focus.

## Verification

- Automated authentication, Rune-payment, official-game, and two-device LAN tests
- ESLint and production Vite build
- Windows x64 one-click installer packaging
- Responsive battle-layout checks for desktop, tablet, and phone widths

## Install or update

Download `Rift-Local-Setup-1.3.4-x64.exe` from this release and run it. Existing desktop installations can use **Updates → Check for updates → Download update → Install & restart** once the installer, blockmap, and `latest.yml` are attached to the same GitHub Release.

Windows may show a SmartScreen warning because this personal build is unsigned. Only allow the firewall prompt on a trusted Private network.

## Rules scope

Rift Local enforces the universal best-of-one Duel flow and the vetted Zaun Warrens trigger. Other printed card abilities still use constrained, logged manual-effect controls because the bundled public catalog does not contain complete structured rules text.

## Known limitations

- Zaun Warrens is the only printed trigger automatically discovered in v1.3.4. Players must identify and order any other simultaneous printed triggers together, then record their results with the manual effect tool.
- The in-app manual effect form submits one agreed operation at a time. Multi-instruction card abilities without a vetted handler are not a fully automated Chain simulation.
- Manual recycle operations are tabletop adjudication tools rather than source-issued server prompts; use them only when printed text instructs a legal recycle.
